# 产品概览

Zenith Admin 是一个面向企业后台、运营平台和内部工具场景的全栈后台管理系统。

## 核心定位

它的定位不是“做一个空白管理台壳子”，而是提供一套已经具备基础管理能力、并且适合继续扩展的项目底座。

适合这些团队：

- 想快速上线后台系统的创业团队
- 需要统一技术栈和代码规范的中小团队
- 需要做客户演示、PoC 或内部运营工具的项目组

## 技术栈

| 层级 | 技术 |
| --- | --- |
| 后端 | Hono v4 + Node.js |
| 前端 | React 19 + Vite 8 |
| UI | Semi Design v2 |
| ORM | Drizzle ORM |
| 数据库 | PostgreSQL |
| 缓存/会话 | Redis（ioredis） |
| 任务调度 | pg-boss |
| 实时通信 | WebSocket |
| 桌面客户端 | Electron |
| 校验 | Zod |
| 认证 | JWT Bearer Token（双 Token） |

## 产品价值

### 降低初始化成本

用户、角色、菜单、字典、文件管理等后台常见模块已经具备基础能力，不必每个项目都重头搭建。

### 内置工作流引擎

可视化流程设计器（审批/抄送/条件分支/延迟器）、表单库与自定义业务表单、业务系统接入（businessKey 关联外部实体）、流程模板、审批代理、流程自动化与事件总线，无需引入外部引擎即可覆盖大多数审批场景。

### 即时通讯开箱可用

基于 WebSocket 的单聊/群聊、消息回复/撤回/编辑/转发、@提及、投票、语音消息、收藏置顶与 WebRTC 通话记录等能力，快捷聊天可在任意页面唤起。

### 多渠道通知体系

站内信、公告推送、邮件（SMTP）、短信（多服务商）均已落地，并配备模板管理与发送日志。

### 安全能力全链路覆盖

JWT 双 Token + 会话黑名单 + 登录失败锁定 + IP 黑白名单 + 数据脱敏 + 幂等防重复提交 + 接口限流，满足企业级后台安全合规要求。

### AI 功能区

内置多服务商 AI 对话（支持 PDF 上传预览）、提示词模板、用量统计与 AI 服务商管理页面，用户可按需配置自己的模型密钥；工程结构配合 Zenith Skill 可通过 AI 指令快速生成 CRUD 模块。

### 支付中心开箱即用

内置多渠道支付通道配置（微信支付 / 支付宝等）、支付订单生命周期管理（下单/查询/关闭/退款）、支付回调日志全链路追踪，可直接接入业务系统。

### 会员体系全覆盖

前台 C 端（`member.html` 独立 SPA）与后台管理系统双轨并行、彻底隔离。后台提供会员管理、等级体系、积分账户、钱包余额（乐观锁防并发超扣）、优惠券模板、发券记录与签到，前台支持手机号验证码、手机号 / 邮箱 / 用户名密码登录与会员自助。

### 数据分析与错误监控

内置行为埋点采集（PV/UV/停留/点击）、指标大盘展示与多维筛选；前端 JS 异常自动上报，错误堆栈一键查看，无需额外接入 APM 工具。

### 一站式系统运维

Web SSH 终端（多分屏）、SSH 配置与 SFTP、终端录屏回放、活动终端会话、服务器文件管理器、进程管理、端口监听、Docker 容器管理、systemd 服务管理、网络诊断——运维操作不必离开管理后台。

### 保持演进空间

`shared` 层统一类型、常量和 schema，适合在新增业务时保持前后端一致性。

### 面向团队协作

接口规范、分页格式、页面布局、时间处理方式等都已经形成约定，适合多人协作开发。
