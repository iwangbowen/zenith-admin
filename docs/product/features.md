# 功能模块

当前项目已经具备一套完整的后台管理基础模块。

## 基础模块列表

### 权限与组织

| 模块 | 说明 |
| --- | --- |
| 用户管理 | 用户 CRUD、状态管理、角色/部门/岗位分配、批量操作（批量删除、批量改状态）、Excel 批量导入（含模板下载）、解锁锁定账号、管理员重置密码 |
| 角色管理 | 角色 CRUD、菜单权限分配（树形勾选）|
| 菜单管理 | 树形菜单、目录/菜单/按钮三种类型、全部展开/折叠 |
| 部门管理 | 树形组织层级维护 |
| 岗位管理 | 岗位信息维护、批量删除 |

### 系统基础

| 模块 | 说明 |
| --- | --- |
| 数据字典 | 字典主表与字典项管理 |
| 系统配置 | 系统行为动态配置（验证码开关、密码策略、登录失败锁定等），详见[系统内置配置](/backend/system-configs) |
| 系统监控 | 服务器运行状态（CPU、内存、Node.js 版本等）|
| 仪表盘 | 统计卡片（用户总数、在线人数、今日登录/操作次数）+ 通知公告摘要 |

### 通知与日志

| 模块 | 说明 |
| --- | --- |

### 行政区划

## 模块特性

### 权限与安全体系

- JWT 鉴权（Access Token + Refresh Token 自动续期）
- 路由访问控制与细粒度按钮权限（基于 Context 编程式控制）
- 角色与菜单权限关联
- 动态菜单路由：根据用户角色自动注册可访问页面
- 会话单点登录与强制下线管控
- 登录失败次数限制 + 账号自动锁定（基于 Redis 计数）
- 密码复杂度校验 + 密码过期强制修改
- 图形验证码（可通过系统配置开关）
- 注册功能开关（可通过系统配置控制）
- IP 白名单/黑名单（可独立启用，支持 CIDR 网段）

### 实时通信

- WebSocket 支持实时通知推送（新通知、系统广播）
- WebSocket 强制下线消息推送（管理员踢人后前端自动退出）
- 前端自动重连机制（连接断开后指数退避重试）
- 健康检查：`GET /api/health` 接口，用于服务探活

### 数据规范

- 列表接口统一返回分页结构 `{ list, total, page, pageSize }`
- 参数使用 `schema.safeParse()` 校验
- 统一响应格式 `{ code, message, data }`
- Zod Schema 前后端共享，避免重复定义

### 文件存储

- `local`：本地文件系统（默认）
- `oss`：阿里云 OSS

通过 `file_storage_configs` 表中的 `is_default` 字段切换默认存储。

### 多租户

- 环境变量 `MULTI_TENANT_MODE=true`（前后端均需设置）开启
- 开启后各业务表按 `tenant_id` 自动隔离，删除租户时级联清理下属数据
- 平台超管可在顶栏切换至任意租户视角查看数据
- 登录时可传入 `tenantCode` 定向进入指定租户
- 默认关闭，关闭时与普通单实例模式完全兼容
