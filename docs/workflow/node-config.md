# 节点配置指南

流程中的每个节点都需要根据业务场景进行配置。点击节点卡片打开右侧配置抽屉，即可配置节点属性。

## 节点类型速览

| 节点类型 | 用途 | 是否生成任务 |
| --- | --- | --- |
| 发起人 | 流程起点，填写表单发起申请 | 否 |
| 审批人 | 需要审批人审批的节点 | 是 |
| 办理人 | 需要执行人办理任务的节点 | 是 |
| 抄送人 | 仅通知相关人员 | 否 |
| 条件分支 | 根据条件选择一条分支执行 | 否 |
| 并行分支 | 同时执行所有分支 | 否 |
| 包容分支 | 执行满足条件的所有分支 | 否 |
| 路由分支 | 按表达式路由到不同分支 | 否 |
| 延迟器 | 延迟一段时间后继续 | 否 |
| 触发器 | 执行 HTTP 请求或数据操作 | 否 |
| 子流程 | 调用另一个流程定义（支持单 / 多实例） | 是 |

## 发起人节点

发起人节点是流程的起点，用户在此填写表单并提交申请。

**可配置项**：
- **节点名称**：如"填写请假单"
- **表单权限**：设置发起人可见、可编辑的表单字段

## 审批人节点

审批人节点是流程中最常用的节点，需要指定人员审批后才能继续。

### 审批人指定方式

| 指定方式 | 说明 |
| --- | --- |
| 指定成员 | 选择一个或多个具体人员 |
| 指定角色 | 角色对应的所有成员（受角色「管理范围」过滤） |
| 部门负责人 | 发起人所在部门的负责人 |
| 指定岗位 | 所选岗位下的所有成员 |
| 部门成员 | 指定部门下的全部成员（可含子部门） |
| 发起人自己 | 流程发起人自行处理 |
| 发起人部门分管领导 | 发起人所在部门的分管领导 |
| 直属主管 | 发起人的直属上级（支持设置层级） |
| 连续多级上级 | 从直属上级开始逐级向上审批，可设置终点条件 |
| 连续多级部门负责人 | 从直属部门负责人逐级向上 |
| 表单联系人 | 取表单中人员字段对应的人员 |
| 表单内部门 | 取表单中部门字段，取该部门负责人 |
| 节点审批人 | 关联前序节点的实际审批人 |
| 用户组 | 指定用户组的成员 |
| 发起人自选 | 发起时由发起人自由选择审批人 |
| 发起人自选(指定范围) | 发起时由发起人在指定范围（成员/角色/部门/用户组）内选择 |
| 审批人自选 | 由上一节点审批人在审批时选择本节点审批人 |
| 流程表达式 | 通过表达式动态计算审批人 ID |

### 审批方式

当审批人节点配置了多人时，需要指定审批方式：

| 方式 | 说明 |
| --- | --- |
| 或签 | 任一审批人通过即节点通过，其余待审批人自动跳过 |
| 会签 | 所有审批人都通过才节点通过 |
| 依次审批 | 按顺序逐一审批，前一人通过后才轮到下一人 |
| 比例会签 | 同意比例达到设定阈值（`approveRatio`，1–100%）即节点通过 |
| 随机一人 | 系统从候选人中随机指派一人审批 |
| 自动通过 | 不生成审批任务，节点自动通过 |

### 审批类型

| 类型 | 说明 |
| --- | --- |
| 人工审批 | 需要审批人手动审批（默认） |
| 自动通过 | 节点自动通过，不生成任务 |
| 自动拒绝 | 节点自动拒绝 |

### 角色「管理范围」过滤

当审批人指定方式为「指定角色」时，系统会按角色配置的「管理范围」（部门列表）过滤实际成员：

- 若角色未配置管理范围，视为全员，行为与旧版本一致；
- 若角色配置了管理范围，仅当用户所在部门（或其下级部门）位于该范围内时，才会被纳入审批人；
- 范围支持选择多个部门，子部门自动包含。

该机制对应飞书「角色管理范围」概念，用于按提交人所在部门动态收敛角色审批人。配置入口：系统管理 → 角色管理 → 编辑角色 → 管理范围。

### 空审批人处理

当审批人解析为空时（如部门负责人不存在、表单字段为空），可选择：

| 策略 | 说明 |
| --- | --- |
| 自动通过 | 节点自动通过 |
| 转交管理员 | 转交给系统管理员处理 |
| 自动拒绝 | 节点自动拒绝 |
| 转交指定人 | 转交给预设的指定人员 |

### 发起人与审批人同一人时的处理

| 策略 | 说明 |
| --- | --- |
| 由发起人自己审批 | 发起人同时是审批人时，自己审批 |
| 自动跳过 | 自动跳过该节点 |
| 转交给直接上级审批 | 转交给发起人的直接上级 |
| 转交给部门负责人审批 | 转交给发起人的部门负责人 |

### 审批人去重

当流程中多个节点的审批人为同一人时：

| 策略 | 说明 |
| --- | --- |
| 自动跳过 | 后续节点自动同意（默认） |
| 仍需审批 | 每个节点都需重新审批 |

### 操作权限

可配置审批人在该节点可执行的操作：

| 操作 | 说明 |
| --- | --- |
| 允许审批通过 | 同意该节点 |
| 允许拒绝 | 驳回该节点 |
| 允许转交 | 将任务转交给其他人处理 |
| 允许加/减签 | 临时增加或移除审批人 |
| 允许回退 | 退回至上一步或指定节点 |
| 允许评论 | 添加审批意见 |
| 审批同意时需手写签名 | 通过时要求手写签名 |
| 提交审批需填写审批意见 | 必须填写意见才能提交 |

### 操作按钮设置

在「操作按钮」配置中，可对每个动作按钮做更精细的定制：

| 按钮 | 可定制项 |
| --- | --- |
| 通过 | 显示名称、意见名称、是否要求上传附件 |
| 拒绝 | 显示名称、意见名称、是否要求上传附件、**跳转目标节点**（驳回时跳到指定节点） |
| 转办 | 显示名称、意见名称、是否要求上传附件 |
| 委派 | 显示名称、意见名称、是否要求上传附件 |
| 加签 | 显示名称、意见名称、是否要求上传附件 |
| 退回 | 显示名称、意见名称、是否要求上传附件、**跳转目标节点** |

> 「通过」「拒绝」按钮始终启用，其余按钮可单独启停；按钮的「跳转目标节点」优先级高于节点级驳回策略。

### 表单权限

设置该节点下各表单字段的可见性和可编辑性：

| 权限 | 说明 |
| --- | --- |
| 可读 | 可看到但不可修改 |
| 可编辑 | 可看到且可修改 |
| 隐藏 | 不可见 |

### 超时处理

可配置节点的超时处理策略：

| 配置项 | 说明 |
| --- | --- |
| 启用超时 | 是否启用超时处理 |
| 执行动作 | 自动提醒 / 自动通过 / 自动拒绝 |
| 超时时间 | 数值 + 单位（分钟 / 小时 / 天）：当超过该时长未处理时触发 |
| 最大提醒次数 | 仅「自动提醒」时有效，按超时时长重复提醒，达到次数后停止 |
| 提醒耗尽后 | 仅「自动提醒」时有效，提醒次数用尽仍未处理时的升级动作 |

**提醒耗尽后（升级处理）**：当执行动作为「自动提醒」且达到最大提醒次数后，可继续执行一次升级动作：

| 升级动作 | 说明 |
| --- | --- |
| 不处理 | 保持任务挂起，等待人工处理（默认） |
| 自动同意 | 系统自动通过该节点 |
| 自动拒绝 | 系统自动拒绝该节点 |
| 转交给上级 | 将任务转交给当前处理人的上级（按「上级层级」取第 N 级部门负责人），并重置提醒计数重新计时 |

> 超时扫描由后台定时任务驱动（默认每 1–5 分钟一次），因此实际触发时间存在一个调度周期的延迟。

### 节点监听器

可在审批/办理节点上挂载 **Webhook 监听器**，在节点触发指定事件时向外部 URL 发起 HTTP 调用（独立于全局「事件订阅」，按节点维护）：

| 配置项 | 说明 |
| --- | --- |
| 触发事件 | `onCreate`（节点任务创建）/ `onApprove`（节点通过）/ `onReject`（节点驳回），可多选 |
| 请求地址 | 目标 URL |
| 请求方法 | `GET` / `POST` |
| 请求头 | 自定义 Header（JSON） |

一个节点可配置多个监听器，分别监听不同事件、投递到不同地址。

## 办理人节点

办理人节点与审批人节点类似，但主要区别在于办理人执行的是"办理"动作而非"审批"动作。

**可配置项**：
- 办理人指定方式（同审批人节点）
- 空办理人处理策略
- 表单权限

## 抄送人节点

抄送人节点仅用于通知相关人员，不产生审批/办理动作。

**可配置项**：
- 抄送人指定方式（同审批人节点）
- 仅同意时抄送：仅在流程通过时发送抄送通知
- 表单权限

## 条件分支

条件分支根据预设条件选择一条分支执行，分支之间是**互斥**的。

**配置要点**：
- 每个**非默认分支**必须配置至少一个条件规则（否则保存时会拦截；空条件分支会被当默认分支抢先命中）
- 条件规则可基于**表单字段值**或**发起人维度**判断
- 支持「且」（所有条件同时满足）和「或」（任一条件满足）两种组合方式
- 始终保留一个默认分支（「其它情况」，不设置条件），作为兜底
- **优先级（匹配顺序）**：分支按从上到下的顺序依次匹配，命中第一个满足条件的分支即停止。可用分支卡片上的 **上移 / 下移** 按钮调整优先级（默认分支始终保持末尾）

**条件来源**：

每条规则可选择两类来源：

| 来源 | 说明 | 可选「字段」 |
| --- | --- | --- |
| 表单字段 | 取表单中某个字段的值进行比较 | 流程表单中的字段 |
| 发起人维度 | 取流程发起人的属性进行比较 | 发起人本人 / 发起人部门 / 发起人角色 / 发起人岗位 |

> 发起人维度在流程发起时对发起人的部门 / 角色 / 岗位做一次快照求值。其中「发起人部门」按**包含子部门**语义匹配：选择某个父部门即可命中其下所有子部门的发起人。

**条件规则**：

| 运算符 | 说明 | 适用 |
| --- | --- | --- |
| 等于 | 字段值等于指定值 | 表单字段 |
| 不等于 | 字段值不等于指定值 | 表单字段 |
| 大于 | 字段值大于指定值 | 数字、日期 |
| 大于等于 | 字段值大于等于指定值 | 数字、日期 |
| 小于 | 字段值小于指定值 | 数字、日期 |
| 小于等于 | 字段值小于等于指定值 | 数字、日期 |
| 包含在 | 字段值在指定列表中 | 文本、选择 |
| 不包含在 | 字段值不在指定列表中 | 文本、选择 |
| 包含 | 字段值包含指定文本 | 文本 |
| 为空 | 字段值为空（未填写） | 表单字段 |
| 不为空 | 字段值不为空（已填写） | 表单字段 |
| 属于 | 发起人属于所选成员/部门/角色/岗位 | 发起人维度 |
| 不属于 | 发起人不属于所选成员/部门/角色/岗位 | 发起人维度 |

## 并行分支

并行分支会**同时执行所有分支**，所有分支都完成后才会继续后续节点。

> 每条并行分支都必须包含至少一个节点（不能为空），否则保存时会拦截——空分支会导致运行时汇聚（join）永远等不到而卡死。

## 包容分支

包容分支会执行**满足条件的所有分支**，所有满足条件的分支都完成后才会继续。

新建包容分支节点时会自动追加一条「其它情况」默认分支，作为兜底支：当所有显式条件分支都不匹配时，流程进入该默认分支，避免漏走。默认分支无需配置条件，名称可重命名但不可删除其默认属性。

> 每条**非默认**分支都必须配置条件且至少包含一个节点；默认分支（「其它情况」）可以为空，作为"无操作"兜底。

## 路由分支

路由分支按表达式路由到不同分支，适用于复杂的分支逻辑。

## 延迟器

延迟器节点可让流程暂停一段时间后再继续。

**配置项**：
- **延迟类型**：固定时长 / 指定日期
- **固定时长**：设置数值和单位（分钟/小时/天）
- **指定日期**：选择具体日期时间

## 触发器

触发器节点在进入该节点时执行预定义动作。详见 [触发器节点](./trigger-nodes.md)。

## 子流程

子流程节点可调用另一个已发布的流程定义，实现流程复用与编排，能力对标钉钉宜搭 / 飞书审批的子流程节点。

### 基础

- **选择子流程**：下拉选择一个已发布的流程定义（自动排除当前流程，避免直接自引用）。
- **调用模式**：
  - **单实例**：调用一次子流程，发起一个子实例。
  - **多实例**：遍历父表单的数组型字段，对每一项各发起一个子实例。
- **等待子流程完成**：
  - 开启（默认，同步）：父流程在子流程结束（通过 / 驳回）后再继续下游节点。
  - 关闭（异步）：父流程不等待，立即继续下游节点（fire-and-forget），子实例独立运行，不汇聚结果。

### 多实例

- **循环数据源**：父表单中的数组型字段（多选 / 复选框 / 标签 / 人员 / 部门）。每个元素发起一个子实例。
- **执行方式**：
  - **并行**：一次性发起全部子实例，全部结束后父流程继续。
  - **串行**：依次发起，前一个子实例结束后再发起下一个。
- **某个子实例被驳回时**：
  - **中止整个节点**：立即按节点驳回策略处理父任务。
  - **忽略并继续其余实例**：忽略该子实例的驳回，继续其余实例。
- **当前循环项写入子表单字段**（可选）：将当前循环项的值写入子实例的指定字段；映射中亦可用 `{{item}}` / `{{item.属性}}` 引用。

### 子实例发起人

- **父流程发起人**（默认）：沿用父流程发起人。
- **取表单字段**：从父表单指定字段（存放用户 ID）解析发起人。
- **指定成员**：固定为指定用户。解析失败时回退父流程发起人。

### 入参 / 出参映射

- **入参映射（父 → 子）**：结构化键值对。左侧选择子流程字段，右侧选择父表单取值；取值支持模板：
  - `{{form.字段key}}` / `{{字段key}}`：引用父实例表单字段（单一引用保留原值类型）。
  - `{{item}}` / `{{item.属性}}`：引用当前循环项（仅多实例）。
- **出参映射（子 → 父）**：左侧父字段，右侧子字段。子实例结束后将子字段值回写父字段；**多实例时按子实例完成顺序聚合为数组**写回。

### 子流程被驳回后

- **忽略驳回，按通过继续**：子流程驳回不影响父流程，按通过推进。
- 否则按节点 **驳回策略** 处理（终止 / 退回上一节点 / 退回发起人 / 退回指定节点）。

> 说明：子实例会写入 `parentInstanceId` / `parentTaskId`，在流程实例详情中可通过「子流程」标签页查看子实例及其状态并跳转，子实例详情亦提供返回父流程实例的入口。子流程支持嵌套（子实例内部可再含子流程节点）。
