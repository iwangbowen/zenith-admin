# Zenith Admin 后台管理脚手架功能规划

## 当前已具备能力

- 认证与登录：JWT 登录、注册、个人中心、自助修改密码
- 基础 RBAC：用户、角色、菜单、角色分配菜单、角色分配用户
- 基础系统管理：字典管理、文件存储配置、文件管理、通知公告
- 运维与审计：服务监控、登录日志、操作日志
- 前后端共享：`@zenith/shared` 统一维护类型、常量和 Zod 校验

## 仍欠缺的功能（按优先级）

### P0：基础必需

1. 用户管理支持管理员修改密码
2. 部门管理（树形组织结构）
3. 用户部门 / 岗位关联
4. 更细粒度的权限点控制（按钮 / API）
5. 前端路由级权限校验

### P1：常用增强

1. 系统配置管理
2. Excel 导入导出
3. 数据备份与恢复
4. 更完整的审计日志（变更前后对比）
5. 消息模板与通知配置
6. 文件预览与分类管理

### P2：体验与效率增强

1. 批量操作
2. 高级搜索与查询条件保存
3. 定时任务管理
4. 个性化工作台配置
5. API 文档生成
6. 操作日志可视化

### P3：平台化扩展

1. 站内信 / 消息中心
2. 工作流 / 审批流
3. 国际化（i18n）
4. 代码生成器
5. 报表 / 数据大屏
6. 实时通知推送

## 本次实现范围

本轮优先补齐以下三项：

1. **用户管理支持修改密码**
   - 在用户管理页增加“修改密码”入口
   - 增加管理员为指定用户重置 / 修改密码接口

2. **增加部门管理**
   - 新增部门表、部门 CRUD 接口、树形部门页面
   - 支持父子部门结构与删除校验

3. **用户部门 / 岗位关联**
   - 新增岗位管理
   - 用户支持单部门归属 + 多岗位关联
   - 用户创建 / 编辑时可选择部门和多个岗位

## 本次落地后的收益

- 后台脚手架具备更完整的组织模型
- 用户维度从“账号 + 角色”扩展为“账号 + 部门 + 岗位 + 角色”
- 可直接作为中后台项目起步模板，而不必从零补组织架构能力

## 后续建议

- 下一阶段优先补齐：按钮 / API 权限、前端路由守卫、系统配置管理
- 如果后续业务更复杂，可继续扩展：多部门归属、岗位分组、数据权限范围
